# v1.1.0 Release Plan

> Scope: Supabase migration lane, MinIO integration alignment, Redis GUI onboarding
> Status date: 2026-04-15

## Cadence

1. `v1.1.0-beta.1` — Foundation and consistency
2. `v1.1.0-rc.1` — Migration rehearsal and rollback verification
3. `v1.1.0` — Production cutover

## Milestones

### v1.1.0-beta.1 (In Progress)

- [x] Normalize docs and scripts to real core service names (`pg-db`) and dev port (`16380`)
- [x] Split environment templates by boundary (`core`, `ml`, `supabase`, `tools`)
- [x] Add RedisInsight overlay (`docker-compose.tools.yml`)
- [x] Add Supabase migration starter overlay (`docker-compose.supabase.yml` with `supabase-meta`)
- [x] Add Makefile entry points (`tools-*`, `supabase-*`)

### v1.1.0-rc.1 (Planned)

- [ ] Create migration checklist from current `pg-db` schema to Supabase lane
- [ ] Dry-run migration on staging clone
- [ ] Verify MinIO object-path compatibility and bucket policy constraints
- [ ] Execute rollback drill (DB restore + application rollback)

### v1.1.0 (Planned)

- [ ] Switch production DB lane to Supabase target
- [ ] Freeze write window and perform final migration
- [ ] Post-cutover validation (`health`, task read/write, export/import, ML prediction)
- [ ] Publish release notes and operator handoff checklist

## Operator Notes

- Keep `.env` focused on current core runtime.
- Keep migration credentials in `.env.supabase` only.
- Keep local tooling settings in `.env.tools` only.
- Do not copy Supabase migration keys into `.env`.
